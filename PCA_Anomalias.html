Detección de Anomalías¶

El enfoque de detección de anomalías (también conocido como valores atípicos o extraños), basado en el análisis de componentes principales (PCA) se ha utilizado con éxito para monitorear sistemas con variables altamente correlacionadas.

En este notebook se propone un esquema de detección de anomalías basado en PCA, sobre un conjunto de datos que representan imágenes de perros y gatos, con la intención de revelar si una imagen particular es un gato o no. Probaremos eventualmente también el caso contrario.

}

El análisis de componentes principales buscará abstraer las características mas relevantes que describen a un gato (o perro). Para lograr una buena abstracción, es necesario normalizar los datos, ya que cada columna (variable o caractrística) de nuestros datos, pueden estar definidas entre rangos de valores muy distintos, donde las variables con mayor rango de variabilidad lograrían opacar a las demás variables.

A continuación se "des-proyectan" los valores asociados a cada imágen, es decir, se re- proyecta la transformación inicial en los ejes originales.

In [14]:
gatos.train.proyec <- (gatos.train %*% matriz.proyec) %*% t(matriz.proyec)

Con la siguiente línea se obtienen los errores asociados a la re-proyección del paso anterior. Si los residuales son pequeños, significa que la imágen corresponde a un gato, en otro caso sería un perro u otra cosa (asumimos perro puesto que la base unicamente tiene perros y gatos)

Proyecciones usando menos componentes¶

Queremos validar cómo se ven afectadas las proyecciones, o lo que es lo mismo, cómo incrementan los residuales, cuando usando menos componentes principales. Vemos a continuación la evaluación de los residuales para todos los conjuntos de datos usando entre 2 y 70 componentes.

In [24]:
componentes <- 2:ncol(pc$x)

pc_residuales <- data.frame(N_CP=NULL, gatos.test.res=NULL, perros.train.res=NULL, perros.test.res=NULL)
for(i in componentes){
     pc_residuales[i-1, 1] <- i
     
     # gatos test
     gatos.test.resid <- gatos.test - ( (gatos.test %*% matriz.proyec[,1:i]) %*% t(matriz.proyec[,1:i]) )
     pc_residuales[i-1, 2] <- mean(apply(gatos.test.resid, 1, norm, type="2"), na.rm=T)
     
     # perros train
     perros.test.resid <- perros.test - ( (perros.test %*% matriz.proyec[,1:i]) %*% t(matriz.proyec[,1:i]) )
     pc_residuales[i-1, 3] <- mean(apply(perros.test.resid, 1, norm, type="2"), na.rm=T)
     
     # perros test
     perros.train.resid <- perros.train - ( (perros.train %*% matriz.proyec[,1:i]) %*% t(matriz.proyec[,1:i]) )
     pc_residuales[i-1, 4] <- mean(apply(perros.train.resid, 1, norm, type="2"), na.rm=T)
}
names(pc_residuales) <- c("Num_CP", "gatos.test.res", "perros.train.res", "perros.test.res")

Clasificación de perros y gatos¶

Bucamos ahora un algoritmo que nos permita clasficar perros y gatos. Para ello aplicamos componentes principales sobre la base de entrenamiento para perros, luego realizamos proyección sobre los datos de prueba y aplicamos el algoritmo k-medias, esperando poder clasificar correctamente las imagenes.

In [29]:
centro.perros <- colMeans(perros.train)

gatos.train <- gatos.train -
               matrix(centro.perros, nrow(gatos.train), ncol(gatos.train), byrow=TRUE)
gatos.test <- gatos.test -
              matrix(centro.perros, nrow(gatos.test), ncol(gatos.test), byrow=TRUE)
perros.train <- perros.train -
                matrix(centro.perros, nrow(perros.train), ncol(perros.train), byrow=TRUE)
perros.test <- perros.test -
               matrix(centro.perros, nrow(perros.test), ncol(perros.test), byrow=TRUE)

## Componentes principales
pc_perros <- prcomp(perros.train)
matriz.proyec <- pc_perros$rotation

## Proyecciones
gatos.test.py <- (gatos.test %*% matriz.proyec) %*% t(matriz.proyec) 
perros.test.py <- (perros.test %*% matriz.proyec) %*% t(matriz.proyec)
perros.train.py <- (perros.train %*% matriz.proyec) %*% t(matriz.proyec)

total.py <- rbind(gatos.test.py, perros.test.py)
py_label <- c(rep(1, 29), rep(2, 29))

Sabemos por la construcción anterior que los primeros 29 registros son gatos y los demás son perros. Nótese que el algoritmo no parece clasificar bien, pues logramos un error del 46.5%.
